Lidar data need to be stored as NetCDF files in the hierarchy folder root/WMO/yyyy/mm/ where
folder root is defined in options.json, and WMO is the station identifier (e.g. 0-20000-0-07617 for the
miniMPL Toulouse mobile system).
Foreachnightthescriptloadsthedailyfilesforthenight-endingdatetheprecedingday,guaranteeing
coverage of the full night. This is performed using load l1 data() from data loader.py. The night is
labelled by the date on which it ends (e.g. the night labelled 15/01/2025 spans from 14/01/2025 to
15/01/2025).

Next, filter cloudy profiles() from data loader.py removes cloud-contaminated profiles. The
script does the following procedure:
1. Discardprofilescontaininglowclouds(altitude< z low cloud,default6000m,definedinoptions.json).
2. Remove all profiles within a ±15 min window around each low-cloud detection.
3. For the remaining high-cloud cases, mask the signal from 500 m below the cloud base upward.
After each step the algorithm checks whether the number of remaining profiles is above the minimum
required duration min time range (hours) from options.json. If it is not the case, the calibration
constant is not computed.
3 Loading atmospheric model (standard or ECMWF)
Depending on the chosen model, the script calls either load standard atmosphere() or load ecmwf
profile() from atmosphere.py.
– Standard atmosphere: Returns temperature and pressure profiles from the US Standard Atmo-
sphere 1976 (file standard atmosphere US 1976 50km.csv) interpolated onto the lidar range grid.
– ECMWF:Retrievesthenearestgridpointusingtheinstrumentlatitudeandlongitude,interpolates
the first time step onto the lidar grid, and fills NaNs at the bottom of the profile with the nearest
valid value.

8 Writing results to NetCDF
Thecalibrationresultsarewritten(orappended)toayearlyNetCDFfilebythefunctionwrite calibration
result() in output.py. The following variables are stored:
– calibration method: identifier of the calibration type (0 for Rayleigh calibration).
– time: date corresponding to the end of the night used for the calibration.
– start time and end time: timestamps of the first and last profiles of the night used.
– lidar constant: the median lidar constant Cmedian.
L
– lidar constant uncertainty: the robust uncertainty 2σ .
robust
– calibration bottom heightandcalibration top height:lowerandupperboundsofthecalibra-
tion window selected by the Rayleigh fit.
– laser wavelength: laser wavelength (determined from the instrument type).
– housekeepingvariables:laser life time,status detector,status laser,temperature optical
module, window transmission, and optical module id.
If results for the same day and the same calibration method already exist, the entry is overwritten.
If the calibration method differs, the new results are appended.
